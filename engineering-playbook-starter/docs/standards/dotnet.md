# .NET Standards

## Architecture

- Использовать Vertical Slice / Feature Slice для application layer.
- Команды и запросы разделять по ответственности.
- Доменные ошибки возвращать через `Result<T, Error>` (не через исключения).

## Code style

- Публичные методы должны быть короткими и явными.
- Один метод — одна причина изменения.
- Не использовать магические строки и числа без констант.

## API contracts

- Любой breaking change только с миграционным планом.
- DTO и доменные модели не смешивать.
- Валидация входа обязательна.

## Data access

- Для сложных reads использовать специализированный query-слой.
- Миграции атомарные, обратимые, с описанием риска.

## Quality gates

- Unit tests для бизнес-правил.
- Integration tests для API и инфраструктурных интеграций.
- Логи и трассировка обязательны для новых endpoint.
