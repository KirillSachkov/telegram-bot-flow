## Coding rules

- Keep changes minimal and focused.
- Follow existing architecture and naming.
- Validate through targeted tests first.
- Avoid speculative refactoring.

## AI workflow

1. Gather context from relevant files.
2. Provide short plan for non-trivial tasks.
3. Implement in small safe steps.
4. Run checks/tests.
5. Report changes, risks, verification.

## Forbidden

- No unrelated code changes.
- No hidden breaking changes.
- No invented APIs or fake facts.
