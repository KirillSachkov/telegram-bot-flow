# Aider Conventions

## Working mode

- Явно указывать файлы в контексте перед запросом.
- Использовать короткие итерации (1 задача = 1 PR).

## Prompt pattern

- Цель
- Ограничения
- Файлы
- Критерии приемки
- Команда проверки

## Safety

- Не принимать большие автогенерированные диффы без review.
- Любой breaking change — отдельный PR с миграционным планом.
