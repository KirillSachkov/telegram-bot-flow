# Cursor Rules Template

Используй как базу для `.cursor/rules/*.mdc`.

## Rule: Scope discipline

- Вносить только изменения в рамках задачи.
- При необходимости затронуть соседние модули — явно объяснять причину.

## Rule: Architecture respect

- Соблюдать принятые слои и границы модулей.
- Не обходить доменные инварианты.

## Rule: Verification

- После изменений запускать релевантные проверки.
- Если проверки недоступны, дать ручной сценарий проверки.
